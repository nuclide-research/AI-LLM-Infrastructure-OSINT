==============================================================================
METHODOLOGY NOTES - IMPROVEMENTS OVER CAT-49
==============================================================================

1. The first verifier run reported 1 unauth_argilla_confirmed. Investigation
   showed the host was running Argilla 1.4.0-dev0 (pre-v1) and returning
   HTTP 200 with text/html on /api/v1/me, the Nuxt SPA was catching any
   /api/v1/* path the backend did not mount and serving the index page.
   The verifier was reading "200" as unauth without validating the JSON
   shape. The fix: require application/json content-type AND a valid
   Argilla user shape ({"username": ...} or {"id": ..., "role": ...})
   before declaring an unauth verdict. The same host's pre-v1 /api/me
   endpoint returned the correct 401 + Argilla auth-error JSON.

2. Added an explicit /api/me fallback. Pre-v1 Argilla (1.0 - 1.13) used
   /api/me, /api/datasets, /api/workspaces without the /v1 prefix. The
   verifier now tries both paths and reports which one answered.

3. Insight candidate (likely Insight #79 if codified):
   **SPA HTML catch-all defeats naive HTTP-200-as-unauth verifiers.**
   Always validate content-type AND a positive shape match before
   declaring an unauthenticated endpoint open. This is a sibling
   methodology to Cat-49's <pre>-strip insight.

THESIS POSITION (vs Cat-49 Label Studio)
==============================================================================

  Label Studio   99.75 % auth-gated   (1/407 LIVE was empty fresh install)
  Argilla       100.00 % auth-gated   (0/23 LIVE returned unauth /me)

  Argilla is the new ceiling on the auth-friction gradient. Both are
  data-labeling / RLHF platforms, the same threat surface (labeled
  training corpora, annotator PII - RLHF preference data), and both
  demonstrate that the "mandatory first-run signup + auth-required
  /api/" pattern is the strongest defensive posture in the entire
  NuClide measurement set.

  The 23-host sample is much smaller than Label Studio's 407 LIVE, so
  the population-level conclusion sits on weaker statistics. Two ways
  to firm this up: (1) widen the dork (ssl.cert.subject:argilla,
  org-cert sweeps, /api/_info marker probes on a CIDR sweep); (2)
  re-run when the next Argilla LTS releases and the Shodan visible-
  population is naturally refreshed.
