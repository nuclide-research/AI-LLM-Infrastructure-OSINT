VULNERABILITY LANDSCAPE
========================================

CVE-2026-44345 — Malicious Bento Package RCE
  Affected: BentoML < 1.4.39 (current stable)
  CVSS: 9.8
  Attack Vector: HTTP POST to /models or /api/v1/models
  Impact: Full RCE as bentoml process user
  Status: EXPLOITABLE on 9/14 verified hosts

CVE-2025-27520 / CVE-2025-32375 — Model Deserialization RCE
  Affected: BentoML < 1.4.8
  CVSS: 9.1–9.8
  Attack Vector: Malicious pickled object in model directory
  Impact: Code execution during model load
  Status: LIKELY EXPLOITABLE on 4/14 verified hosts

CVE-2025-54381 — Path Traversal via Archive Extraction
  Affected: BentoML < 1.4.37
  CVSS: 7.5
  Attack Vector: Tar member paths not validated in safe_extract()
  Impact: Arbitrary file write outside model directory
  Status: RELEVANT to model upload chains

CVE-2024-2912 / CVE-2024-9070 — Pickle RCE in Runner Server
  Affected: BentoML < 1.3.4post1
  CVSS: 9.8
  Attack Vector: Runner server accepts pickled objects without validation
  Impact: Code execution on runner processes
  Status: MAY BE EXPLOITABLE if runner server is accessible

METHODOLOGY CODIFIED
========================================

Insight Candidate: Auth-Off-By-Default Prevalence Varies by Adoption & Deployment
  Evidence: BentoML ships with access_authorization=False hardcoded, yet population
           is small (~14 confirmed hosts) on the public internet
  Hypothesis: Large-scale platforms achieve small public footprints through:
    1. Defense-by-obscurity (limited public deployment)
    2. Defense-by-isolation (internal networks only)
    3. Vendor-imposed auth layers (BentoCloud, Yatai reverse proxy)
  Impact: Population-scale findings require verification; Shodan dork FP rate is
          load-bearing (80.3% on BentoML)

Dork Quality Assessment:
  "http.title:BentoML" — HIGH false positive rate (captures Grafana, Node.js)
  Recommendation: Use HTTP endpoint verification (Layer 7) over TCP banner matching
  Improvement: Build BentoML-specific marker in /docs.json (x-bentoml-name header)
