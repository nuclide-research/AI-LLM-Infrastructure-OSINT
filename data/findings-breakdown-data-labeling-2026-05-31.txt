THESIS RESULT (the headline)
----------------------------
Auth-on-default HOLDS for the category, so the finding is the DEFAULT-OPEN KNOB:
  Label Studio: open self-registration ON by default (DISABLE_SIGNUP_WITHOUT_LINK=False)
                -> 16 of 17 confirmed hosts have /user/signup reachable = effective-unauth
                (anyone registers -> full /api/projects + /api/tasks raw labeled data). 1 closed.
  CVAT:         registration OFF by default (OPA model since 2.0.0)
                -> 20 of 20 confirmed AUTH-ON (/api/projects 401/403). 0 effective-unauth.
  doccano:      ALLOW_SIGNUP=False default -> 1 confirmed AUTH-ON (consistent w/ 2026-05-04 348/348).
=> The discriminator is the DEFAULT VALUE of the registration knob. Same hosts/clouds, opposite
   outcomes. Insight #72.

INSIGHTS CODIFIED
-----------------
#72 ships-auth-but-default-open-registration: a platform that ships authentication AND a real authz
    layer can still be effective-unauth if its self-registration knob defaults open. Discriminator =
    the registration default. Label Studio (open) 16/17 vs CVAT/doccano (closed) hold.

    /api/server/about return 404/406 to a generic application/json probe; 0/30 without the vendor
    media type, 20/30 with it. Fingerprint discovery must send the platform's content-negotiation header.
