WHY 8 FALSE POSITIVES?
================================================================
Shodan body-text dork `"chainlit"` matches any HTML containing
the string. Hits include:
  - "BioStar 2" access-control web UI (54.147.124.167) — string
    appears in a CSS class or asset filename, not Chainlit.
  - Apache 404 pages, ASP.NET request-rejected pages,
    "Fusion AIQ" SaaS landing page, biomass brokerage UI.
  - 157.90.251.84 serves a vite SPA shell with Chainlit-template
    structure but all og:* identity stripped — likely a Chainlit
    fork/whitelabel; treated as FP for the survey because no
    canonical marker survives.

Lesson: the Chainlit dork has a ~42% body-text FP rate. The
markers that survive are og:url -> github.com/Chainlit/chainlit
and og:image -> chainlit-cloud.s3.eu-west-3.amazonaws.com. Both
are baked into the upstream index.html template and only fall
away when an operator deliberately edits the template.

================================================================

  / , /auth/config , /user , /project/translations
No POST. No /message. No chat-history reads. No file uploads.

`/user` was probed without authentication; a 200 response means
the server accepts anonymous sessions, not that any chat thread
was opened or any prompt sent. Project names ("Renovis", "POC AI

================================================================
CANDIDATE INSIGHT
================================================================
Cat-50 Chainlit reinforces the auth-on-default thesis (Insight
#40) WITH AN INVERSION: Chainlit's framework default is auth-OFF.
The framework ships with no authentication unless the operator
adds `@cl.password_auth_callback` or OAuth env vars. The 80%
unauth rate at population scale (4/5 verified hosts) is what
"auth-off-default" looks like with no countervailing pressure.

Compare:
  Ollama (Cat-04):        auth-off-default, ~95% unauth (huge pop)
  Weaviate (Cat-02):      auth-off-default, dominant unauth tier
  LangFuse (Cat-15):      auth-on-default for newer releases
  Chainlit (Cat-50):      auth-off-default, 80% unauth (tiny pop)

The pattern holds: framework default sets the policy at population
scale. Operators do not flip the switch.
