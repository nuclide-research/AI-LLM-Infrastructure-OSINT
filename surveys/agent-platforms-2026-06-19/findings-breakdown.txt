THESIS / INSIGHT CANDIDATES
--------------------------------------------------------------------------------
  C1. Agent-platform category is auth-OFF-by-default behind a dev/prod toggle.
      The auth-on-default thesis (Insight #40, OSS auth strengthens under disclosure
      pressure) does NOT yet hold here: these are research-grade tools that ship the
      open default and disclaim production use. A category-level negative result.
  C2. Provider-key-in-open-agent is the dominant loot, not data. 52/59 hosts leak a
      billable upstream credential surface; the agent IS the key. LLM-jacking >> PII here.
  C3. Version-blind CVE: when the only version oracle is the exploit sink itself,
      severity is indeterminate under a restraint ethic. The finding is the attack
      surface, not the patch state. (Candidate methodology insight.)
  C4. http.title dorks select a high-liveness subpopulation (87% live vs the ~29%
      banner-cache norm) but also collide on generic titles (Function Dashboard,
      OpenHands catch-alls) - conjunctive data-layer markers are mandatory.
  C5. JSON-API agent platforms are Shodan-BANNER-DARK BY CONSTRUCTION. When a platform's
      only discriminating fingerprint lives in a non-root JSON endpoint (/info,
      /openapi.json, /v1/health/, /features), Shodan's crawl-/-only model never captures
      it: port populous, banner empty. Darkness is a property of the CRAWLER'S FETCH
      MODEL, not the operator's network bind - Letta's docker default is 0.0.0.0:8283
      yet it is invisible. Corollary: brand-in-root-SPA platforms (AutoGen, OpenHands,
      SuperAGI) ARE Shodan-selectable; JSON-API-only platforms route to Censys /
      active-probe. Round-1's flat "Shodan-dark" label conflated two distinct mechanisms
      (localhost-bind vs banner-dark); round-2 with primary-source markers separates them.
      This is the chainsaw card-07 lesson at category scale: the marker was wrong, not
      the population.
