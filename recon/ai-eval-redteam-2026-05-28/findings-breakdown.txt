=== FINDING #162 ===

TARGET: evals.dev.generalwisdom.com (100.50.218.102)
PLATFORM: Promptfoo (LLM evaluation / red-team framework)
SEVERITY: MEDIUM (verified)
STATUS: Open

WHAT IS EXPOSED:
The Promptfoo eval server at evals.dev.generalwisdom.com is running with
no authentication. Anyone on the internet can access it.

Specifically, without any login:
- The full list of 60 LLM providers configured for evaluation is readable.
  This includes Anthropic's Claude Haiku, Sonnet, and Opus model family,
  Azure GPT-4o, and 55 other configured providers with their temperature
  and token settings.
- Active eval datasets are readable, including regression test cases linked
  to a file called "regression-core.yaml," the most recent eval run on
  2026-05-01, token usage stats (213,085 tokens consumed in that run), and
  the full evaluation score history (32 pass, 0 fail).
- The operator identity probe (/api/user/email) returns {"email":null},
  which is the Promptfoo indicator for a server running without login
  required.

The Azure provider config shows "your-resource-name.openai.azure.com" as
the API host — this is a placeholder value, so Azure credentials are not
hardcoded in what is readable.

WHY THIS MATTERS:
Promptfoo is specifically used to run adversarial test suites against LLMs.
If this instance is used for red-teaming (which the Anthropic model lineup
and provider count suggests), the adversarial prompt library, test criteria,
and model comparison results would be in the eval history accessible without
credentials. Even without full eval content, knowing which model variants
are being compared and at what parameters is operational security information.

The dev subdomain suggests this is a staging or development environment.
That does not reduce the exposure — dev eval servers often run the most
sensitive pre-production tests.

HOW TO FIX:
Option 1: Add PROMPTFOO_AUTH_USER and PROMPTFOO_AUTH_PASSWORD environment
variables when running Promptfoo (basic auth built into the server).
Option 2: Put the server behind a reverse proxy (nginx, Caddy) with an
authentication layer before it reaches the internet.
Option 3: Bind to 127.0.0.1 instead of 0.0.0.0 if internet access is not
needed.

EVIDENCE:
- curl https://evals.dev.generalwisdom.com/api/user/email -> {"email":null}
- curl https://evals.dev.generalwisdom.com/api/providers -> 60 providers JSON
- curl https://evals.dev.generalwisdom.com/api/datasets -> eval dataset with
  regression-core.yaml test suite, 2026-05-01 eval run, 213K token usage
