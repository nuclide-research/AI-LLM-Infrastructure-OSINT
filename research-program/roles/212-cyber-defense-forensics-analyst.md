# NICE 212 — Cyber Defense Forensics Analyst

**Source PDF:** `~/Documents/dod-cyber-pathways/212-Cyber-Defense-Forensics-Analyst-Career-Pathway.pdf`

**Role definition (NICE):** "Analyzes digital evidence and investigates computer security incidents to derive useful information in support of system/network vulnerability mitigation."

**OPM Occupational Series alignment (top 3):**
- 2210-Information Technology (INFOSEC) — 85%
- 1550-Computer Science — 4%
- 1811-Criminal Investigation — 4%

**Common Work Role Pairings (top 3):**
- 511-Cyber Defense Analyst — 39%
- 531-Cyber Defense Incident Responder — 18%
- 211-Forensics Analyst — 11%

## Core Tasks

| Task ID | Task Statement |
|---|---|
| T0027 | Conduct analysis of log files, evidence, and other information to determine best methods for identifying the perpetrator(s) of a network intrusion. |
| T0036 | Confirm what is known about an intrusion and discover new information, if possible, after identifying intrusion via dynamic analysis. |
| T0075 | Provide technical summary of findings in accordance with established reporting procedures. |
| T0103 | Examine recovered data for information of relevance to the issue at hand. |
| T0167 | Perform file signature analysis. |
| T0286 | Perform file system forensic analysis. |
| T0432 | Collect and analyze intrusion artifacts (e.g., source code, malware, system configuration) and use discovered data to enable mitigation of potential cyber defense incidents within the enterprise. |

## Research-program relevance

Closer to NuClide's operational discipline than 211: digital evidence analysis tied to system/network vulnerability mitigation, OPM 2210 INFOSEC alignment. Tasks T0027 (log analysis), T0240 (network traffic capture), T0432 (intrusion artifact collection), T0400 (incident correlation) map directly to post-disclosure activity once a NuClide finding becomes an active incident at the disclosed party. Reverse-engineering and malware-analysis skills (K0183, K0188, S0131) overlap with VisorAgent adversarial work in the controlled-target lane. Indexed as the natural handoff role when NuClide hands a finding to the affected org's IR/forensics function.
