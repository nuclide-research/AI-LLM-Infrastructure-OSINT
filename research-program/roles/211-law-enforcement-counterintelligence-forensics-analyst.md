# NICE 211 — Law Enforcement Counterintelligence Forensics Analyst

**Source PDF:** `~/Documents/dod-cyber-pathways/211-Law-Enforcement-Counterintelligence-Forensics-Analyst-Career-Pathway.pdf`

**Role definition (NICE):** "Conducts detailed investigations on computer-based crimes establishing documentary or physical evidence, to include digital media and logs associated with cyber intrusion incidents."

**OPM Occupational Series alignment (top 3):**
- 1811-Criminal Investigation — 83%
- 510-Accounting — 7%
- 2210-Information Technology — 3%

**Common Work Role Pairings (top 3):**
- 132-Target Network Analyst — 79%
- 212-Cyber Defense Forensics Analyst — 15%
- 221-Cyber Crime Investigator — 4%

## Core Tasks

| Task ID | Task Statement |
|---|---|
| T0027 | Conduct analysis of log files, evidence, and other information in order to determine best methods for identifying the perpetrator(s) of a network intrusion. |
| T0048 | Create a forensically sound duplicate of the evidence (i.e., forensic image) that ensures the original evidence is not unintentionally modified, to use for data recovery and analysis processes. Includes hard drives, floppy diskettes, CD, PDA, mobile phones, GPS, and all tape formats. |
| T0439 | Detect and analyze encrypted data, stenography, alternate data streams and other forms of concealed data. |
| T0471 | Document original condition of digital and/or associated evidence (e.g., via digital photographs, written reports, hash function checking). |
| T0087 | Ensure chain of custody is followed for all digital media acquired in accordance with the Federal Rules of Evidence. |
| T0103 | Examine recovered data for information of relevance to the issue at hand. |
| T0113 | Identify digital evidence for examination and analysis in such a way as to avoid unintentional alteration. |
| T0165 | Perform dynamic analysis to boot an "image" of a drive (without necessarily having the original drive) to see the intrusion as the user may have seen it, in a native environment. |
| T0167 | Perform file signature analysis. |
| T0286 | Perform file system forensic analysis. |
| T0168 | Perform hash comparison against established database. |
| T0287 | Perform static analysis to mount an "image" of a drive (without necessarily having the original drive). |
| T0179 | Perform static media analysis. |
| T0285 | Perform virus scanning on digital media. |
| T0190 | Prepare digital media for imaging by ensuring data integrity (e.g., write blockers in accordance with standard operating procedures). |
| T0212 | Provide technical assistance on digital evidence matters to appropriate personnel. |
| T0075 | Provide technical summary of findings in accordance with established reporting procedures. |
| T0532 | Review forensic images and other data sources (e.g., volatile data) for recovery of potentially relevant information. |
| T0241 | Use specialized equipment and techniques to catalog, document, extract, collect, package, and preserve digital evidence. |
| T0289 | Utilize deployable forensics tool kit to support operations as necessary. |

## Research-program relevance

Law-enforcement-specific role aligned to OPM 1811 (Criminal Investigation). Not directly invoked in NuClide's AI/LLM infrastructure recon program — operates downstream of confirmed criminal activity with Federal Rules of Evidence chain-of-custody requirements. Indexed for completeness; would map onto post-disclosure handoff when an exposed AI/ML asset turns out to be evidence in an active investigation. Forensics tradecraft (T0027 log analysis, T0048 imaging, T0240 network traffic capture) overlaps with VisorLog ledger discipline conceptually but operates under judicial admissibility constraints NuClide's recon work does not.
