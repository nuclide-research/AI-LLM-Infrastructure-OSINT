INSIGHTS — codified for this survey
================================================================================

  #88 GENERALIZATION CONFIRMED
    Scrape-topology = operator org chart extends from the metrics-substrate tier
    (Cat-46c VictoriaMetrics, Cat-46d Prometheus) to the trace tier (Jaeger
    /api/services, Zipkin /api/v2/services). The same data class — operator's
    internal microservice catalog with named services — leaks through the
    discovery endpoint of every traced-services platform we measured.

  CANDIDATE INSIGHT #95 — Temporal admin-claim window as verification state
    SigNoz's "first registrant becomes admin" creates an auth state that is
    neither "unauth" nor "auth" but "claimable-right-now." Verification primitive
    must read a state field (setup_completed), not just probe an endpoint.
    Population stats are valid only at the moment of measurement; the open
    window can close between candidate harvest and verification run.

  CANDIDATE INSIGHT #96 — Storage backend co-deployment chain prediction
    Predicted (per OSINT) but not verified this run: storage backends (ES 9200,
    ClickHouse 8123, Cassandra 9042) co-deployed unauth on hosts with unauth
    tracers. Scanner banner pass identified 56 ClickHouse and 6 Elasticsearch
    on tracer-corpus IPs; correlation pass deferred to follow-up.

  CANDIDATE INSIGHT #97 — Dork FP-class for short-text body filters
    Tempo /api/echo (4-byte body), Zipkin /config.json conjunct, OTel self-metrics
    /metrics body, Tempo /api/search/tags JSON — all Shodan-blind via http.html:
    body filters. The crawler doesn't index short plaintext or sparse JSON
    consistently. These platforms are only discoverable via port-direct or via
    full-handshake scanner (Step 0c) — confirms scanner's standing/non-skippable
    posture (Insight #77) on this tier.
