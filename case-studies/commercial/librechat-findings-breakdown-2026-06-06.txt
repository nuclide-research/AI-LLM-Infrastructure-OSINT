════════════════════════════════════════════════════════════════════
LLM10 UNBOUNDED CONSUMPTION CLASS
════════════════════════════════════════════════════════════════════

Every open-registration LibreChat instance enables LLM10 attacks: a
registered user invokes the operator's configured LLM provider,
billed to the operator's API key. LibreChat supports OpenAI,
Anthropic, Google, Azure, Bedrock, custom OpenAI-compatible.

This makes LibreChat REGISTRATION_OPEN more economically severe than
Langfuse SIGNUP_OPEN despite the lower percentage rate. The 412 open
LibreChat instances represent 412 potential Denial-of-Wallet targets.

FP CATALOG
════════════════════════════════════════════════════════════════════

1. Some LibreChat hosts default to appTitle "LibreChat" with no
   customization — these are likely lab / test deployments where
   the operator never even branded the install. Severity is not
   reduced (still LLM10-class), but operator attribution is harder.

2. buildInfo.branch == "main" includes both fresh Docker :latest
   builds AND older clones from main pre-v0.8. The 32.7% rate
   represents the union; not a clean signal.

3. UCB host (169.229.156.181:3080) returns the default "LibreChat"
   appTitle — could be: (a) lab deployment, (b) abandoned student
   project, (c) actually configured but using the default brand.
   Disclosure path is still valid (default branding doesn't change
   the operator-attribution).
