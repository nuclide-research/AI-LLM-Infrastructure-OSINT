FP CATALOG
════════════════════════════════════════════════════════════════════

1. LiteLLM model names are NOT provider identity — operators alias
   commercial model names to local Ollama/OpenRouter free models.
   VERIFY via /model/info litellm_params.api_base, not /v1/models id.
   Example: 51.255.95.37 — "claude-3-5-sonnet-20241022" routes to
   ollama/qwen2.5-coder:32b @ localhost.

2. LiteLLM healthy=0 instances are config-disclosure, not key-theft.
   Unhealthy backends = likely rotated/expired keys. Still a finding
   but downgrade from CRITICAL to MEDIUM.

3. Langfuse signUpDisabled=false alone is MEDIUM, not CRITICAL.
   Actual trace data access requires registration + org membership.
   Severity depends on whether org allows all users to see all traces.

4. Kong port 8001 http.title:"Kong" — FP rate TBD. Many Kong deployments
   run behind nginx reverse proxy; :8001 not always the admin API.
   aimap fingerprint verifies via /node_id field in response body.
