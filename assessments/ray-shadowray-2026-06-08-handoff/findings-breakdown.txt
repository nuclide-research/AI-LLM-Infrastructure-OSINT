METHODOLOGY LESSON
  Initial probe used /api/cluster_status — returned 0/2,000 unauth
  Corrected probe   used /api/jobs/      — returned 1,798/2,000 unauth
  Picking the wrong path gives a confident zero. The Jobs API is
  the actually-exposed ShadowRay primitive; the Cluster Status API
  is auth-gated on current Ray versions.

INSIGHT CANDIDATE
  "Population framing inflation by attacker port-fanout": the
  175,189 'exposed Ray' figure overcounts attacker infrastructure
  by ~5.5x because each attacker IP runs Ray on dozens of ports.
  The true attacker-IP count in our sample is ~463 / unique sampling
  of 1,808 = 25.6%. Extrapolating to Shodan pop: ~8,100 attacker
  IPs / ~2,100 clean operator IPs visible behind the 175k surface.

RESTRAINT
  - GET /api/jobs/ only. No POST. No driver_info reads. No
    /api/jobs/<id>/logs. No worker-node interaction.
  - 463 likely-attacker IPs OBSERVED, not engaged.
  - Substrate provider routing (AWS abuse@) is the legitimate
    intervention path; we do not contact operator boxes.
