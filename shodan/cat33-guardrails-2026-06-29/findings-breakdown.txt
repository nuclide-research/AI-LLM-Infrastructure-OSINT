INSIGHT CANDIDATES
============================================================

Cand #I-G (null guardrail class): A guardrail running with models not loaded
reports is_valid=true for all inputs -- false confidence is worse than no
guardrail. LLM Guard -1.0 sentinel = model not initialized = bypass by
design. The /scan/ production path is silently broken while /analyze/ path
works: operators calling the standard API get null protection with no error.

Cand #I-H (Prometheus host-header attribution): LLM Guard /metrics leaks
all Host headers seen by the server as http_server_name labels. Enables
operator attribution without querying WHOIS -- data plane self-reports the
operator brand in real time. Same scanner fingerprints (146.56.180.42,
51.158.248.122, api.ipify.org) appear across multiple LLM Guard instances,
enabling cross-instance correlation without active probing.

Cand #I-I (OSS guardrail auth gap mirrors infra gap): All 4 major OSS
guardrail HTTP frameworks (LLM Guard, NeMo, Vigil, Guardrails AI) default
to no auth. The same auth-on-default failure class that dominates AI/ML
infra has reached the safety layer itself. The defense is undefended.

Cand #I-J (config enumeration as bypass pre-positioning): /v1/rails/configs
(NeMo) and /guards (Guardrails AI) expose loaded policy IDs unauthed.
An attacker maps the rail topology, then crafts inputs targeting the
unloaded categories -- not cracking the guard, finding the gaps in it.

Cand #I-K (guardrail system prompt = security policy disclosure): When a
guardrail model's Ollama backend is internet-accessible, GET /api/show
verbose=true returns the complete SYSTEM block -- which IS the classification
policy. Exposing it reveals bypass vectors that the policy itself documents
(UnisGuard: "安全知识问答→无风险" is in the system prompt). Reading a WAF
ruleset equivalent.

Cand #I-L (dual-layer infrastructure exposure): Operators harden the
application API layer (port 5000 with auth, port 8000 with /check) but
leave the underlying model server (Ollama :11434) internet-exposed. The
guardrail stack is bypassed entirely by calling the infrastructure layer
directly. Architectural pattern: the defense is implemented in middleware
but the backend is left accessible -- same failure class as JWT-authed
gateways sitting in front of unauthenticated internal APIs.
