Summary
-------
Insight #97 promoted from CANDIDATE to NUMBERED. Three independent measured
cohorts of the same backend signature (mcp-server 1.0.1 + 5-tool credential-theft
surface + 2025-06-18 protocol) on the same operator AWS multi-region
infrastructure, across two ports and two time-snapshots, all return I/N at the
deception extreme (>=0.83). Boundary condition codified: I/N applies to HTTPS
cohorts; HTTP-only cohorts fall through to Insight #30.

Operator infrastructure observations
------------------------------------
1. 66/66 original cred-theft IPs still alive at the 24-hour mark with identical
   backend + identical tool surface. Operator infrastructure is stable.
2. 22 net-new IPs match the same backend signature on the same or adjacent port.
   None overlap with the original 66. Operator is expanding, churning, or both.
3. 1 of the original 66 IPs also serves on port 3001 with the identical backend.
   Operator runs the same canned MCP service on multiple ports for at least one
   host. Multi-port serving is the boundary between Insight #30 (multi-port
   discriminator) and Insight #97 (cert distribution discriminator) and reinforces
   their complementarity.
4. All 22 new IPs are AWS multi-region (3.x, 13.x, 15.x, 16.x, 18.x, 43.x, 47.x,
   51.x, 54.x, 56.x prefixes). Same operator profile as the original 66.

   86 unique IPs.
