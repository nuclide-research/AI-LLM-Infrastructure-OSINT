============================================================
INSIGHT CANDIDATES
============================================================

Cand #I-A (two-level auth failure): MLflow auth defaults are doubly broken -- off by default
AND default creds when enabled. No other surveyed platform has this compound failure.

Cand #I-B (CVE cascade from auth gap): 7 CVEs 2024-2026 ALL require no auth. Root cause
is API design: endpoints were built without auth assumption, then CVEs were retrofitted as
input validation fixes rather than auth gates.

Cand #I-C (artifact_location as attack primitive): The file:// artifact_location write
primitive (F2) is not a CVE -- it is a design feature being used maliciously. MLflow
allows custom artifact storage paths, including arbitrary filesystem paths. This is
documented in MLflow's own artifact-store docs.

Cand #I-D (port-5000 registry collision): Port 5000 is used by both MLflow (tracking)
and Docker Registry. Dork-based harvest conflates the two. Delta = 4 hosts (2.5% FP rate
on strict dork). VisorCAS signature candidate: title-absent + Registry API fingerprint.

Cand #I-E (shared-backend attribution): The gateway/chat-* experiment pattern + shared
base58 model IDs across 12+ hosts suggests a single ML platform operator with multiple
public-facing frontend nodes. The entire cluster shares one attack surface.

Cand #I-F (no prior population survey): No published count of exposed MLflow instances
found in OSINT platoon research. NuClide is first. Estimated 62% of live indexed
instances are unauthenticated; ~180-200 unauth instances at any given time.
