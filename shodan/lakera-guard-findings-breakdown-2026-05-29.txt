E. REUSABLE INSIGHT (codify)
   For an invisible server-side dependency, fingerprint the DEPLOYABLE it ships in (template shell),
   the GATEWAY it rides in (LiteLLM/Portkey/Kong config + exposed admin), or the MISCONFIG it
   surfaces — never the dependency. The demo/SE/integrator population is findable (shared template
   shell); bespoke production is not (no shared shell + server-side call). Attribution for bespoke
   customers = marketing/code, not Shodan. A vendor-plugin-catalog string finds exposed admin APIs,
   not vendor usage ("available in build" != "configured").
